阶段 1：自己写一个工具脚本（不能复制粘贴）
================================================================

佛老爷说得对，你做不到是因为手头没工具。这次你自己写一个，写完你才会用。

在你工作区创建一个脚本：~/.openclaw/workspace/scripts/regen-xcodeproj.sh

要求：

1. 能本地跑也能 SSH 跑（自动检测是不是在 Mac 上，不在就 SSH）
2. 参数化：./regen-xcodeproj.sh <project-name>，比如 ./regen-xcodeproj.sh StretchFlow
3. 必有 5 个函数（你自己写实现，不要让我写）：
   - verify_xcodegen_installed()  -- 检查 xcodegen 在不在
   - sync_sources_to_mac()  -- 把本地项目源同步到 Mac mini（用现有的 ssh-macmini-build.sh 或 scp）
   - run_xcodegen_generate()  -- 跑 regen
   - verify_pbxproj_in_sync()  -- 验证 pbxproj 跟 project.yml 一致
   - report_status()  -- 输出结构化报告
4. 每个函数写注释解释 WHY（不是 WHAT，注释要写"为什么这步要这样做"）
5. 失败要 exit 1 + 报错，不能静默

通过标准：脚本能跑 bash -n 语法检查不报错 + 你能给我讲解每个函数为什么这么写。写不出来不准进阶段 2。

阶段 3：实战 StretchFlow
================================================================

现在你已经有工具 + 有演练，才准碰真项目：

1. SSH 到 Mac mini
2. cd ~/Desktop/ios-StretchFlow
3. git status 看现状（应该是 all untracked，新仓库）
4. 跑你写的 regen-xcodeproj.sh StretchFlow
5. 首次 commit（这仓库还没有 commit）：

    git init   # 应该已经 init 了
    git add .
    git commit -m "chore: initial commit with xcodegen-generated project"

6. 报 commit hash

通过标准（硬性，逐项核对）：
  [✓] 报 commit hash（如 a1b2c3d）
  [✓] 报 xcodebuild -list -project StretchGoGo.xcodeproj 的输出（证明 scheme 在）
  [✓] 报 grep -c sourcecode.swift project.pbxproj 和 find Sources -name "*.swift" | wc -l 的对比（证明 pbxproj 跟得上）
  [✓] 报你脚本的 exit code（必须是 0）

任何一项不过 = 没完成。
