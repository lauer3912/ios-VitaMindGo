## 💡 学到的教训

1. **cron job 模型白名单会过期**: 创建时合法的模型, allowlist 更新后会被静默 reject. 需要定期 audit
2. **失败要有 fallback 通知**: cron 投递失败, 主人 session 至少要收到一条 "cron X 失败 N 次" 提醒. 现在没有这个机制, 7 天才发现
3. **`openclaw cron edit --patch` 不存在**: 用 `cron` 工具的 `update` action + 嵌套 patch 对象
4. **CLI 走 scope 升级流程时改用工具**: 工具走的是不同权限通道, 不会被 pairing 卡住
5. **老爷的元规则是黄金法则**: "能做的尽快做" 一句话就让 Agent 从 "汇报" 模式切到 "执行" 模式. 应刻入 SOUL.md 长期生效

   - 证据: 99% 成功 run 都是 NO_REPLY (无内容整合), 8h 太频繁
2. **提高 failureAlert 阈值**: `after: 2` → `after: 4` (2 次连续错误 → 4 次)
   - dreaming cooldown: 1h
   - 8:00 早报 cooldown: 2h (间隔 24h, 2h 防止刷屏)
3. **手动 run 验证 API**: 75s 后仍然 `overloaded` (consecutiveErrors 14 → 15)
   - 确认: **API 真的持续过载**, 降频是对的方向, 但 consecutiveErrors 需等 API 恢复才清零

- allowlist 只有 `minimax/MiniMax-M3`, 临时加 google/anthropic/openai 需要 API key, 没配置

### 教训 (加永久)

1. **环境敏感路径要 3 个平台都核对**: workspace (Mac) / Mac mini (目标机) / 新 Ubuntu (root). 错一个就全错
