## Light + Dark 双主题实现 (15:00-15:30)

佛罗多老爷要求 iOS App 必须支持 Light + Dark 双主题。

### 改造方案

**1. 新增 `Sources/Core/Theme/DynamicColor.swift`**
```swift
extension UIColor {
    convenience init(light: UIColor, dark: UIColor) {
        self.init { trait in
            trait.userInterfaceStyle == .dark ? dark : light
        }
    }
    convenience init(lightHex: String, darkHex: String) { ... }
}

extension Color {
    init(lightHex: String, darkHex: String)
    init(light: UIColor, dark: UIColor)
}
```

**2. 重构 VitaTheme**
- Chrome tokens (background / surface / text*) → 双值 dynamic
- Brand tokens (primary / secondary / accent) → 固定（身份色）
- Card rarity tokens (Common..Legendary) → 固定（卡牌艺术永远深色）

**3. VitaPocketApp 改造**
- 移除 `.preferredColorScheme(.dark)`
- 新增 `@AppStorage("appearanceMode")` + `AppearanceMode` enum (system/light/dark)
- `preferredColorScheme(AppearanceMode(rawValue:).colorScheme)` 生效

**4. SettingsView 新增 Appearance section**
- 三选项 + 选中对勾
- 存 UserDefaults

**5. 全项目替换硬编码 .white**
- HomeView / CollectionView / HabitsView / CoachView / GameCardView
- `.white` → `VitaTheme.Colors.textPrimary`
- `.white.opacity(0.7)` → `textSecondary`
- `.white.opacity(0.5)` → `textTertiary`

### 设计原则
- **Chrome 跟随主题**（页面背景、文字、卡片底色）
- **品牌色不变**（紫/青/金 — App 身份）
- **卡牌艺术保持深色**（GameCardView — 卡牌游戏设计惯例）
